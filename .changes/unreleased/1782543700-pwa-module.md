type: Added
title: pwa module — service-worker update lifecycle + install detection
---

A new `pwa` subpath export (`@niclaslindstedt/oss-framework/pwa`): `usePwaUpdate`, a singleton hook that registers the service worker (via the optional `workbox-window` peer dependency), tracks the precache download as a 0..100 `progress`, and flips `needRefresh` when a new build is waiting; `UpdateToast`, a presentational "a new version is ready" prompt driven by that state (injectable `labels`, centres over the content band via the sidebar's `--app-content-*` insets); and `isStandaloneMobile` / `useStandaloneMobile` to detect an installed PWA on a phone. The service-worker build, the `base`/`cacheId`, and the dev gate are the app's, threaded through `usePwaUpdate`'s config.
